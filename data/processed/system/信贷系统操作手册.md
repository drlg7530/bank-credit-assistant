1. 概要

目标：为信贷业务人员与系统开发提供完整、可执行的功能设计与操作流程，降低业务误操作、缩短培训时间、并支持 RAG/知识库接入（供业务人员随时查询政策与产品说明）。

主要用户角色：

客户经理（营销/受理）

受理岗（录入/初审）

风控/审核岗（反欺诈、征信、授信审批）

放款岗（资金复核、放款）

贷后岗（催收、风险监控）

系统管理员（参数、字典、用户权限）

2. 模块总览（功能清单）

登录与单点工作台（Dashboard）

客户管理（客户档案、联系人、证件）

3. 详细功能设计与操作流程（关键场景，逐步模拟点击）
场景 A：客户经理受理并提交新贷款申请（前端流程示例）

页面路径（菜单）模拟点击：

登录页面 -> 输入账号/密码 -> 点击【登录】

顶栏菜单：点击【工作台】 -> 左侧菜单点击【受理管理】 -> 子菜单【新增贷款申请】

页面：新增贷款申请（表单）

顶部：客户基本信息（客户类型：个人/企业 单选）

表单字段（示例）：

客户姓名/公司名（必填）

证件类型+证件号码（身份证/统一社会信用代码，必填）

联系电话（必填，格式校验）

贷款产品（下拉，来自产品字典）

贷款金额（数字，>0）

期限（下拉/月数）

利率（自动根据产品带出，可手动调整，需审批）

抵押/担保信息（可选）

附件上传（身份证、合同、合同扫描件，支持多文件）

交互与校验（当用户点击“下一步/保存草稿/提交”）：

点击【保存草稿】：前端校验必填字段，提示未填项（红色提示），保存到“草稿”表。

点击【提交审批】：前端做完整校验 -> 若通过，则弹出“确认提交审批”弹窗（显示关键摘要） -> 点击弹窗【确认】后提交到流程引擎并生成申请单号（格式：LN + yyyyMMdd + 6位序号）。

成功后跳转到【申请详情】页，顶部显示流程进度条（受理 -> 风控 -> 审批 -> 放款 -> 结清）。

页面元素（按钮与控件命名建议）：

操作按钮：{保存草稿, 提交审批, 打印申请单, 上传资料, 取消}

状态标签：{草稿、待风控、风控通过、审批中、已放款、逾期、结清}

权限控制：

客户经理仅能【保存草稿】【提交审批】；不能直接放款或修改审批结论。

场景 B：风控岗进行征信与评分（后台操作）

导航：工作台 -> 风控管理 -> 待风控列表 -> 点击某条【查看/处理】

操作步骤（模拟点击）：

在待风控列表，点击申请单号（或行右侧【处理】按钮） -> 打开“风控详情”页（左侧显示申请信息，右侧为操作面板）

风控操作面板包含：征信查询（按钮）、反欺诈检查、评分卡分数、规则引擎建议、人工备注框

点击【征信查询】 -> 系统调用征信接口（显示查询结果摘要：信用分、逾期记录、异议提示）

系统自动跑评分卡并显示【评分：XX】，并给出“准入判断”（通过/拒绝/人工复核）和规则命中明细（例如：命中“个人逾期6个月内” -> 建议拒绝）

风控岗可点击【强制通过】【退回修改】【拒绝】三类操作（每个操作必须填写审批意见）。若选择【强制通过】需触发二次审批（会签流程）。

点击【通过并提交审批】 -> 流程流向审批岗。

关键设计点：

征信查询需要记录调用日志（接口成功/失败、时间、操作人）并展示给审批岗。

规则命中需要可导出明细（便于合规审计）。

任何人工越权（如强制通过）要触发更高权限的复核。 （这些是银行级风控需求的常见做法）。

场景 C：审批岗多级审批（含回退与会签）

导航：工作台 -> 审批中心 -> 待审批列表 -> 点击【审批】

审批页要素：

申请摘要（金额/期限/借款人/评分）

风控建议（自动显示）

可查看所有附件（合同、证明）

审批意见输入框（必填）

审批操作按钮：{同意、不同意、退回补件、会签发起}

会签流程：

如果达到阈值或产品配置要求（例如 >100万 或 风控评分低于阈值），审批人点击【发起会签】，选择会签组（法务/合规/技术），系统创建并行审批子任务，所有会签通过后才能最终通过。

回退/补件：

审批人可选择【退回补件】，系统将申请状态回到“补件中”，并自动在客户经理工作台创建待办任务。

场景 D：放款与打款（含第三方对接）

导航：工作台 -> 放款管理 -> 待放款指令 -> 选择一条 -> 点击【发起放款】

流程：

放款岗在页面确认放款账户（借款人银行账户）、放款金额、放款时间（立即/定时）

点击【生成放款指令】 -> 系统生成放款指令并发送给资金支付系统/第三方支付（或银行核心）

支付回调成功后，系统自动更新申请为【已放款】，并记录交易流水号。若回调失败，系统重试并发起人工复核。

安全与审批：

放款必须有两人复核（制单人/复核人）流程；复核通过后才发送指令（双人权限）。

场景 E：还款与账务（客户自助与线下）

菜单：工作台 -> 还款管理 -> 还款记录 / 生成账单

功能点：

系统按放款产生还款计划（等额本息/等额本金/一次性还本付息等），并发送账单通知（短信/APP/邮箱）

客户可线上还款（点击【立即还款】-> 跳转支付页面 -> 支付完成回调）

支付成功后，系统结清/预还/提前还款按规则计算罚息/违约金并展示明细。

场景 F：贷后监控与催收

导航：贷后 -> 逾期列表 -> 选择客户 -> 催收记录

功能点：

系统按日计算逾期等级（逾期1-30天、31-90、>90），并生成催收任务与催收脚本（标准话术）。

催收人员在催收详情页记录通话、承诺还款计划，系统支持自动提醒与任务分配（工作台提醒）。

对高风险账户触发法律/外包催收流程。

6. 权限矩阵（简略）

客户经理：新增申请、上传附件、查看本人申请、接收待办

受理岗：编辑客户信息、初审、提交风控

风控岗：征信、评分、风控结论（可触发二次审批）

审批岗：审批/会签/退回

放款岗：生成放款指令、复核、查看支付回调

贷后岗：催收、诉讼、减值处理

管理员：字典、产品配置、流程配置、用户权限

7. UI/交互细节（减少误操作）

每一步关键操作（提交、放款、强制通过）均弹确认框并记录操作人/时间/意见。

在关键页面显示“操作提示/校验规则”，例如“当客户有近2年内逾期，风控会自动建议人工复核”。（可参考银行需求文档的提示条设计）。

8. 审计与合规要点

所有审批动作、接口调用（征信、支付）、数据修改都必须写审计日志（who/what/when/old_value/new_value）。

存证保留期与隐私合规（身份证、征信数据）需按监管要求加密与访问控制。
